## Overview

Implement the `testkit` package following the approved design. The package provides two testing modes:
- **Black-Box (`Game`)**: Launches game in separate process, controls via RPC
- **White-Box (`Mock`)**: Tests game logic in-process with mocked inputs

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Socket path collisions in parallel tests | Use unique socket paths per test via AUTOEBITEN_SOCKET |
| Process cleanup fails | Use `t.Cleanup()` for guaranteed cleanup, SIGKILL fallback |
| Game startup slow | Configurable timeout via `WithTimeout()` option |
| Reflection panics on nil values | Check `IsValid()` before accessing fields |
